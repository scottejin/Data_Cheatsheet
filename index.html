  • pandas is for loading, cleaning, filtering, grouping, and preparing data.
  • matplotlib is the underlying plotting library. It is dependable and flexible.
  • seaborn sits on top of matplotlib and makes many statistical plots easier and prettier.
Best beginner rule: if you are in an exam and feel stuck, start with pandas for cleaning, then use either df.plot(...) or seaborn for the graph.

2. A simple exam workflow

  1. Load the data with pd.read_csv() or pd.read_excel().
  2. Inspect it using head(), info(), describe(), and isna().sum().
  3. Clean it by fixing missing values, duplicates, wrong data types, and messy text.
  4. Pick the right graph based on the question being asked.
  5. Label everything clearly: title, axes, legend.
  6. Explain what the graph shows in plain language if the exam asks for interpretation.
Good habit: never jump straight into graphing. A graph based on dirty data can be misleading even if the code runs perfectly.

3. Clean data before graphing

Many graphing problems are actually data cleaning problems in disguise. Keep this section simple: inspect, fix, and only then plot.

3.1 Quick low-level cleaning flow

  1. Check what is missing.
  2. Decide what to drop (critical rows) and what to fill (non-critical fields).
  3. Fix data types (dates, numbers, booleans).
  4. Clean text spacing/casing.
  5. Handle obvious invalid values (e.g., impossible ranges).

Start with a quick health check

1df.head()
2df.info()
3df.isna().sum()
Line-by-line explanation
  1. L1Shows the first few rows so you can quickly inspect what the data looks like.
  2. L2Shows column names, data types, and non-null counts.
  3. L3Counts missing values in each column.
Simple rule: if a key identifier or key date is missing, drop that row. If an optional value is missing, fill it.

3.2 What to drop vs what to fill

Usually drop: missing IDs, missing join keys, rows with no target value when the exam asks to model/compare that target.

Usually fill: optional numeric columns, optional categories, and short gaps in time-series fields.

1# drop rows missing critical id/date
2df = df.dropna(subset=["student_id", "exam_date"])
3
4# fill numeric column with median
5df["score"] = df["score"].fillna(df["score"].median())
6
7# fill category column with label
8df["class"] = df["class"].fillna("Unknown")
Line-by-line explanation
  1. L1Comment: for key fields, missing rows are often safer to remove.
  2. L2Removes rows where student_id or exam_date is missing.
  3. L3Blank line for readability.
  4. L4Comment: median is a simple robust fill for numeric columns.
  5. L5Fills missing score values with the median score.
  6. L6Blank line for readability.
  7. L7Comment: text categories can use an explicit placeholder.
  8. L8Fills missing class values with the text "Unknown".

Extension case: time-series gaps (simple)

1df = df.sort_values("exam_date")
2df["attendance"] = df["attendance"].ffill()
3df["attendance"] = df["attendance"].bfill()
Line-by-line explanation
  1. L1Sorts by date first so forward/backward fill follows time order.
  2. L2Forward-fill: uses the previous known value for gaps.
  3. L3Backward-fill: fills any remaining gaps from the next known value.

3.4 Change data types safely

In beginner datasets, values look numeric/date-like but are stored as text. Convert them explicitly.

1df["exam_date"] = pd.to_datetime(df["exam_date"], errors="coerce", dayfirst=True)
2df["score"] = pd.to_numeric(df["score"], errors="coerce")
3df["attempts"] = pd.to_numeric(df["attempts"], errors="coerce").astype("Int64")
4df["passed"] = df["passed"].str.lower().map({"yes": True, "no": False})
Line-by-line explanation
  1. L1Converts exam_date text to real dates; invalid values become missing (NaT).
  2. L2Converts score values to numbers; invalid text becomes missing (NaN).
  3. L3Converts attempts to nullable integer type (Int64) so missing integers still work.
  4. L4Normalizes text and maps yes/no into booleans for easier filtering.
Important: always use errors="coerce" when cleaning unknown data so bad values become missing instead of crashing your script.

3.5 Basic text cleanup

Text inconsistencies can split one category into many fake categories (e.g., "Math", " math ", "MATH").

1df["name"] = df["name"].str.strip().str.title()
2df["email"] = df["email"].str.strip().str.lower()
3df["subject"] = df["subject"].str.replace(r"\s+", " ", regex=True).str.strip()
Line-by-line explanation
  1. L1Removes extra spaces and title-cases names (e.g., "jANE " -> "Jane").
  2. L2Removes spaces and lower-cases emails for consistency.
  3. L3Reduces repeated whitespace to single spaces, then trims edges.

Extension case: convert placeholder text to real missing values

1placeholders = ["", " ", "NA", "N/A", "-", "unknown"]
2df = df.replace(placeholders, np.nan)
Line-by-line explanation
  1. L1Defines common fake-missing values often seen in CSV files.
  2. L2Replaces all placeholder values with proper missing values (NaN).

Currency and percentages stored as text

  1. L1Removes dollar signs and commas from currency text.
  2. L2Converts cleaned fee text into numeric values.
  3. L3Blank line for readability.

Simple range checks before plotting

1df.loc[(df["score"] < 0) | (df["score"] > 100), "score"] = np.nan

Line-by-line explanation
  1. L1Flags impossible score values (outside 0-100) and marks them missing.

3.7 Outliers

Outliers are unusually high/low values. They are not always wrong. Keep them unless you have a clear reason to remove or cap them.

3.8 Mini checklist before any graph

  • No critical missing keys left.
  • Numeric columns are actually numeric.
  • Date columns are datetime type.
  • Categories are standardized (same spelling/case/spacing).
  • Obvious invalid values handled.

14. Common mistakes beginners make

Using the wrong graph
Example: using a pie chart or line chart when a bar chart would be clearer.
Not cleaning data first
Missing values, wrong types, and duplicate rows can distort the result.
No labels
A graph without a title or axis labels is much harder to understand.
Overcomplicating the graph
If a simple bar chart answers the question, do not make a fancy multi-layer plot.

Extra caution about pie charts

Pie charts exist, but they are often harder to compare accurately than bar charts. If your exam specifically asks for a pie chart, use one. Otherwise, a bar chart is often the better choice.
